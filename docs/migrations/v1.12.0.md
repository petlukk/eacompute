# Migration to v1.12.0

Template / staging file for the v1.12.0 release. Items listed here are
**planned** — they will be filled in as the corresponding intrinsic
deprecations land on `main`.

See [`README.md`](README.md) for the deprecation-cycle policy.

## Summary (planned)

- Monomorphic rename for `sat_add` / `sat_sub` / `abs_diff` — polymorphic
  forms remain functional but emit deprecation warnings.
- `widen_u8_u16_hi` lands as a sibling intrinsic (additive, no break).
- `cvt_f32_f16` gains a width-16 form via AVX-512 (additive, no break).
- `lo_i16x8` / `hi_i16x8` lane extractors land (additive, no break).

The breaking-removal cycle for the polymorphic names will complete in
**v2.0.0** — they remain callable throughout v1.12.x and v1.13.x with the
warning.

## Breaking changes (deprecation warnings, not removals)

### Item: `sat_add` / `sat_sub` / `abs_diff` → monomorphic spellings

- **Why:** the project standard is "concrete, not generic" — every other
  intrinsic carries its element type in the name (`pack_sat_i16x8`,
  `wmul_u32`, etc.). The polymorphic three are remnants from before that
  rule was codified. Making the spelling carry the type makes error
  messages, autocomplete, and search-and-replace land where the caller
  expects.
- **Deprecation cycle:** warning emitted from v1.12.0; removal scheduled
  for v2.0.0.
- **Migration recipe:**

  ```ea
  // Before:
  let x = sat_add(a, b)       // a, b: i8x16
  let y = abs_diff(p, q)      // p, q: u16x8

  // After:
  let x = sat_add_i8x16(a, b)
  let y = abs_diff_u16x8(p, q)
  ```

  Concrete spellings to be added:
  - `sat_add_i8x16`, `sat_add_u8x16`, `sat_add_i16x8`, `sat_add_u16x8`,
    `sat_add_i32x4`, `sat_add_u32x4`
  - `sat_sub_i8x16`, …, `sat_sub_u32x4`
  - `abs_diff_i8x16`, …, `abs_diff_u32x4`

  Plus the wider variants (`*x32`, `*x16`) where the underlying op exists.

- **Affects:** `Olorin` (uses `sat_add` in several places), `eachacha`
  (uses `abs_diff`). Grep `.ea` files in consumers before tagging v2.0.0.

## Additive changes (no migration needed)

- `widen_u8_u16_hi(u8x16) -> u16x8` — partner to existing `widen_u8_u16`
  (which returns the low half).
- `cvt_f32_f16(f32x16) -> f16x16` — width-16 form (AVX-512 only,
  `--avx512` gated). Closes asymmetry with `cvt_f16_f32`.
- `lo_i16x8(i16x16) -> i16x8`, `hi_i16x8(i16x16) -> i16x8` — the lane
  family currently exposes lo/hi for i8/u8/i32/f32 but not i16; needed
  to make the i16 dot-product recipe ergonomic.

## Other v1.12.0 changes

- Deprecation-warning infrastructure itself (this migrations directory,
  `DeprecationWarning` type, `check_types_with_warnings`).
- `cargo public-api` CI gate to catch unintentional library-surface
  changes.
